
웹 어플리케이션#
 90년대 후반부터, 인터넷이 보급화 되면서 발달한 프로그램분야로써,
 그당시 기존에 만들어지던 운영체제에 설치되서 작동하던 툴방식의 프로그램보다
 사용자 입장에서 쓰기 쉽고, 접근용이하다는 장점을 가지고 발달하게 된다.
 (운영체제에 설치된 웹브라우저를 이용해서 사용하면 되니까.)
 
 웹프로그램의 장점은 개발자입장에서도 여러가지 면이 있는데, 
 클라이언트가 사용할 툴을 안만들어도 되고, 
 기능추가나, UI 구축및 변경도 쉽다. 
  (웹 프로그램은 사용자측 UI가 HTML 이라는 언어로 구축이 되는데,
   서버측에서 HTML을 전송해주는 방식이기 때문에..)
 

#웹 프로그램을 만드려면 ?#
 현재와서는 다양한 언어로 만들수 있지만, 크게 3가지 방식이 존재했음. 
  ASP - C기반 
  JSP/servlet - JAVA기반 
  PHP - 
   
#자바로 만들어내는 웹프로그램의 장점?#
 - 타언어와 비교해 보았을때 월등히 풍부한 라이브러리를 활용할수 있다.
 - 쓰레드 형태로 작동되어, 구동되는데 있어서 많은 스펙이 요구되지 않는다.
 - 다양한 환경에서 가동된다는 자바 특유의 장점이 적용된다.


#웹 프로그램을 만들고 구동해보려면?#

 - JDK 
 - 웹 어플리케이션 제작모드의 툴 (Eclipse)
   (기존에 사용중이던 Eclipse 모드는 웹 프로그램 생성이 안됨 / 
    업그레이드 : help > market place > search를 키워드는 web / eclipse project로 두고 검색
    혹은 생성가능한 버전을 설치  JAVA EE Developer 버전 )
 - WAS (Web Application Server)
   : 자바로 만들어낸 웹프로그램을 구동시켜주는 서버인데, 
    사용할껀 Tomcat 이라는 서버인데, tomcat 외에도 Jboss, Resin, Jeus 등 다른 서버들도 존재한다.
    참고,오라클에서 배포하는 WAS는 글래스피시 라는 서버이다.
    (톰캣 서버 다운 :  tomcat.apache.org 에서)
      인스톨버전 혹은 zip 버전 받아서 사용
